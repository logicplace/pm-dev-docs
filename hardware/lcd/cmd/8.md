# ADC Select

* To write columns left to right, use:
  * libpmdd: `LCD_ADC(NORMAL);`
  * C raw: `LCD_CTRL = 0xA0;`
  * ASM: `LD LCD_CTRL, #0A0h`
* To write columns right to left, use:
  * libpmdd: `LCD_ADC(REVERSE);`
  * C raw: `LCD_CTRL = 0xA1;`
  * ASM: `LD LCD_CTRL, #0A1h`

You must set <abbr title="Address Column">ADC</abbr> Select, aka Segment Driver Direction Select, prior to writing to the display. Changing it after writing to the display does not change the visuals.

Because the display RAM extends past the width of the LCD itself, reversed ADC will display different columns than normal ADC.

| ADC     | Visible | Not visible |
| ------- | ------- | ----------- |
| Normal  | 0~95    | 96~131      |
| Reverse | 131~36  | 35~0        |

Because the [LCD Controller](../../cpu/lcd_controller.md) will still write to 0~95 even in reversed ADC, it's possible to have a portion of the screen which is directly driven by software and a portion which is not, possibly allowing a left-hand menu or something which uses 4+ colors while the controller rendering is active.
